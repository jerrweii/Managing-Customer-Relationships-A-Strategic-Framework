# Skill 08: Interaction Design

## Purpose

Design customer interactions so each one either serves the customer, learns something useful, or both.

## When To Use

- Designing communication workflows.
- Building support, sales, success, or marketing interactions.
- Creating interaction history and follow-up logic.

## Inputs

- Channel map.
- Customer journey.
- Interaction types.
- Feedback signals.
- Employee workflows.

## Repeatable Steps

1. Define the purpose of each interaction.
2. Capture the context before, during, and after the touchpoint.
3. Record customer intent, outcome, and follow-up commitments.
4. Feed interaction learning back into the profile.
5. Avoid asking for information the system already knows.

## Software Artifacts

- Interaction timeline.
- Conversation history.
- Follow-up task model.
- Feedback capture.
- Next-best-action context.

## Acceptance Checks

- Interactions update customer memory.
- Employees can see prior relevant context.
- Follow-ups are tracked to completion.
- The customer experience improves over repeated interactions.
