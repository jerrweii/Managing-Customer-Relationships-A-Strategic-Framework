# Skill 05: Customer Data Stewardship

## Purpose

Treat customer data as entrusted information that must be protected, governed, and used responsibly.

## When To Use

- Designing data collection.
- Building CRM integrations.
- Handling consent, privacy, compliance, or audit requirements.

## Inputs

- Customer data inventory.
- Consent requirements.
- Data sources and destinations.
- Retention policies.
- Security and compliance constraints.

## Repeatable Steps

1. Identify what customer data is collected and why.
2. Map each field to a purpose and retention rule.
3. Define who can access, change, export, or delete data.
4. Capture lineage from source to downstream use.
5. Add auditability to sensitive data actions.

## Software Artifacts

- Data inventory.
- Consent model.
- Data lineage map.
- Retention policy implementation.
- Access and audit controls.

## Acceptance Checks

- Every sensitive customer field has a purpose.
- Consent and data use are enforceable in software.
- Data access is logged where risk requires it.
- Downstream systems cannot bypass trust controls.
