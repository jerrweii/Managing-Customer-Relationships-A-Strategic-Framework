# Skill 03: IDIC Process Design

## Purpose

Turn the IDIC model into repeatable CRM system modules: Identify, Differentiate, Interact, Customize.

## When To Use

- Designing CRM architecture.
- Breaking a CRM roadmap into modules.
- Evaluating whether a CRM feature is core or peripheral.

## Inputs

- Customer identity requirements.
- Customer value model.
- Needs and preference model.
- Interaction channels.
- Personalization goals.

## Repeatable Steps

1. Identify: recognize the customer across channels and records.
2. Differentiate: rank or segment by value and needs.
3. Interact: capture meaningful dialogue and feedback.
4. Customize: adapt treatment based on what is known.
5. Connect all four steps through explicit data contracts.

## Software Artifacts

- IDIC capability map.
- Module boundary diagram.
- Event flow from identification to customization.
- API contracts between CRM modules.
- IDIC coverage matrix.

## Acceptance Checks

- Each IDIC step has a responsible module.
- Data flows forward from recognition to treatment.
- Customer learning loops back into future interactions.
- The system can explain why a customer received a specific treatment.
