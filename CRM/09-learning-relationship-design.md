# Skill 09: Learning Relationship Design

## Purpose

Build CRM behavior that remembers, adapts, and improves after each customer interaction.

## When To Use

- Designing personalization.
- Building customer memory.
- Reducing repeated questions or generic treatment.

## Inputs

- Interaction data.
- Preferences.
- Customer outcomes.
- Treatment history.
- Feedback and corrections.

## Repeatable Steps

1. Capture what was learned in each interaction.
2. Store learning in durable customer memory.
3. Use learning to change future treatment.
4. Make personalization explainable to employees and customers where needed.
5. Measure whether learning improves customer outcomes.

## Software Artifacts

- Customer memory model.
- Preference history.
- Learning event stream.
- Personalization rules.
- Treatment effectiveness metrics.

## Acceptance Checks

- The system remembers useful customer context.
- Learning changes future interactions.
- The system avoids repetitive customer effort.
- Customer memory has governance and correction paths.
