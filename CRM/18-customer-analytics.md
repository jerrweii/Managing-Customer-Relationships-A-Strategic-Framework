# Skill 18: Customer Analytics

## Purpose

Use analytics to decide which customers to acquire, grow, retain, serve differently, or stop over-investing in.

## When To Use

- Building predictive CRM features.
- Designing customer segmentation.
- Prioritizing sales, service, or retention actions.

## Inputs

- Profile data.
- Interaction history.
- Transaction history.
- Product usage.
- Outcomes such as churn, expansion, satisfaction, or cost.

## Repeatable Steps

1. Start with the decision analytics will support.
2. Define target outcomes and prediction windows.
3. Build features from customer behavior, value, need, and interaction data.
4. Validate model performance and bias risks.
5. Operationalize scores through workflows and measure lift.

## Software Artifacts

- Churn model.
- Propensity model.
- Next-best-action score.
- Customer health score.
- Model monitoring dashboard.

## Acceptance Checks

- Analytics output maps to a CRM action.
- Scores are refreshed on an appropriate cadence.
- Users can see confidence or explanation where needed.
- Model performance is monitored after deployment.
